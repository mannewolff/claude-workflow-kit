## Kontext
Das GitHub-GraphQL-Kontingent (5.000 Punkte/Stunde) erschoepft sich bei intensiver Board-Nutzung regelmaessig (am 2026-07-06 live gemessen: 0/5.000 verbleibend, REST unberuehrt bei 5.000/5.000). Ein Verursacher: GitHubIssueTracker._ensureProjectMeta in kit/board.mjs holt Project-ID, Status-Field-ID und Option-IDs via gh project list + gh project field-list bei jedem CLI-Aufruf neu, obwohl diese Werte praktisch nie aendern. Der In-Memory-Cache (this._projectId) hilft nicht, weil jeder board.mjs-Aufruf ein neuer Prozess ist.

Hinweis: kit/board.mjs wird mit dem board-ui-Repo gesynct (siehe Commit 4a88999) — die Aenderung muss dort nachgezogen bzw. zurueckgesynct werden.

## Aufgabe
In kit/board.mjs, Klasse GitHubIssueTracker:
- _ensureProjectMeta liest zuerst eine persistente Cache-Datei (z.B. .claude/board-meta-cache.json im Projektverzeichnis, Schluessel: owner + projectNumber). Nur bei Cache-Miss werden gh project list / field-list aufgerufen und das Ergebnis in die Cache-Datei geschrieben.
- Invalidierung: Wenn ein item-edit mit gecachten IDs fehlschlaegt (z.B. Option-ID existiert nicht mehr), Cache verwerfen, Meta neu laden, Operation einmal wiederholen.
- Cache-Datei in .gitignore des Zielprojekts aufnehmen (Installer/Doku entsprechend ergaenzen).

## Akzeptanzkriterium
- Zwei aufeinanderfolgende Aufrufe von `node kit/board.mjs issue list --status backlog` (issueTracker: github): Beim zweiten Aufruf finden nachweislich keine gh project list / field-list Calls mehr statt (Nachweis z.B. via GH_DEBUG=api).
- Loeschen der Cache-Datei erzwingt beim naechsten Aufruf den Neuaufbau; danach existiert die Datei wieder mit gueltigen IDs.
- Ein moveIssue mit absichtlich verfaelschter Cache-Datei (ungueltige Option-ID) heilt sich selbst (Invalidierung + Retry) statt mit Stacktrace zu scheitern.
- board.mjs bleibt als Single-File-Tool lauffaehig ohne weiteren Repo-Kontext; der Cache-Pfad wird relativ zum Arbeitsverzeichnis des Zielprojekts aufgeloest.

## Abhängigkeiten
Keine. (Beruehrt denselben Codebereich wie das Issue "Gezielter Item-Lookup" — Merge-Reihenfolge beliebig, aber nacheinander umsetzen.)
