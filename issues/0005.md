## Kontext
`tools/version.mjs` (Issue #0003) liest die Config mit `JSON.parse` und schreibt sie mit `JSON.stringify(config, null, 2)` zurueck. Dadurch werden beim Bump auch unveraenderte Strukturen neu formatiert — beim ersten realen Patch-Bump (v1.4.1) wurde z.B. das einzeilige `buildChecks`-Array auf mehrzeilig aufgeklappt. Ein Versions-Bump soll ausschliesslich die Versionskennung aendern, nicht die Formatierung der uebrigen Config.

## Aufgabe
- In `tools/version.mjs` den Schreibpfad umstellen: statt die geparste Config neu zu serialisieren, nur den Versions-String im Roh-Text der Datei per gezieltem Regex-Replace auf `"version": "x.y.z"` ersetzen. Die Validierung der bestehenden Version (Format x.y.z) bleibt erhalten; schlaegt der Replace fehl, sauberer Fehler + Exit 1.
- `.claude/workflow.config.json`: das beim v1.4.1-Bump aufgeklappte `buildChecks`-Array wieder einzeilig herstellen (Ausgangsformat).

## Akzeptanzkriterium
- `node tools/version.mjs --patch` aendert im Diff ausschliesslich die eine `version`-Zeile; keine andere Zeile/Formatierung wird beruehrt.
- `--minor`/`--major`/`--get` und die Fehlerfaelle (ungueltige/fehlende Version, kein Flag, keine Config) verhalten sich unveraendert (Exit 1, Config bei Fehlern unveraendert).
- `tools/version.mjs` bleibt lauffaehig ohne weiteren Repo-Kontext (Single-File, nur `node:*`-Imports).

## Abhängigkeiten
Baut auf Issue "Version-Bump-Helfer + version-Feld in der Config" (#0003) auf.
