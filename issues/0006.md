## Kontext

Beim Verbindungstest (#1) schlug node .claude/kit/board.mjs issue list fehl:

Unknown flag: --state.


Der GitLab-Adapter baut in listIssues() das Kommando glab issue list --state opened --output json. Die installierte glab-Version (1.107.0) kennt kein --state-Flag. glab listet standardmaessig offene Issues; ein Filter laeuft ueber --all bzw. --closed. Anlegen und Verschieben von Issues funktionieren, nur das Listen ist betroffen.

## Aufgabe



        
      Datei: .claude/kit/board.mjs, Methode listIssues(status), aktuell Zeile 499.

        
      
--state opened aus dem Kommando entfernen (offen ist der glab-Default). Ergebnis: glab issue list --output json, ergaenzt um das bestehende --label-Handling.

        
      Pruefen, ob an weiteren Stellen im GitLab-Zweig --state verwendet wird.

## Akzeptanzkriterium



        
      
node .claude/kit/board.mjs issue list gibt die offenen Issues als Liste zurueck (u.a. #1), ohne Fehler.

        
      
node .claude/kit/board.mjs issue list --status backlog filtert korrekt nach dem Backlog-Label.

        
      Kein --state-Flag mehr im an glab uebergebenen Kommando.

## Abschlussbericht Issue #0006

### Änderungen
- `kit/board.mjs` — `GitLabIssueTracker.listIssues()`: `--state opened` aus dem `glab issue list`-Kommando entfernt (glab 1.107.0 kennt das Flag nicht mehr, offen ist bereits Default). Kein weiterer `--state`-Gebrauch im GitLab-Zweig gefunden (geprüft per grep).
- `install.mjs` — eingebetteter BOARD_MJS_B64-Blob per `tools/sync-blobs.mjs` neu synchronisiert.

### Tests und Checks
- `node --check kit/board.mjs` → Syntax OK.
- Live gegen echtes lokal installiertes `glab 1.107.0`: `glab issue list --state opened --output json` reproduziert exakt den gemeldeten Fehler (`Unknown flag: --state.`, Exit 1). Mit der reparierten Kommandozeile (`glab issue list --output json`) tritt dieser Fehler nicht mehr auf (nur noch der erwartete Repo-Fehler außerhalb eines Git-Repos).
- `glab issue list --help` bestätigt: kein `--state`-Flag vorhanden, offene Issues sind Default.
- `node tools/sync-blobs.mjs --check` → Blobs synchron.

### Hinweise
- Keine weiteren Stellen im GitLab-Adapter betroffen.
- `--status backlog`-Filterung (zweites Akzeptanzkriterium) hängt vom `--label`-Handling ab, das unverändert blieb und in Issue #0007 separat adressiert wird (Label-Netting-Bug bei moveIssue).
